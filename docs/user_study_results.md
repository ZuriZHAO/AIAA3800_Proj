## 3. 假设判定

- **H1 情绪感知准确**（Q1）：中位 4、均值 3.88，87.5% 打 4–5 → **支持**（A-P04 感知冲突为唯一反例）。
- **H2 音乐符合心情**：ToM 心情贴合中位 4、均值 3.75 → **支持趋势**。
- **H3 ToM 更贴心**（盲评）：ToM 5 : Std 2 : tie 1 → 62.5% → **边际支持**。ToM 在需求贴合/帮助度/疲劳贴合均优，唯 **音乐质量** Std 更高（曲风更平稳、偏钢琴音色）。
- **H4 情绪正向变化**：效价平均 +1.0（5/8 正向）、疲劳多数下降、Q4 中位 4 → **支持**。
- **H5 隐私可接受**（Q5）：均值 4.13、中位 4.5 → **可接受，无强抵触**。
- **H6 疲劳贴合**：整体 fatigue_fit 中位 4；高疲劳子集（P3-1/P3-2）方向正确，样本仅 2 人。

## 4. 定性反馈（按主题归纳）

把每人各条反馈拆开、按主题编码。括号内为来源被试。

### 正面

**A · 情绪感知能力**（4 人提及，最集中的认可点）
- “能感受到我表层情绪下的更深一层，很丰富细致的情感感知。”（A-P01）
- “识别情绪很到位。”（A-P02）
- “比较贴合我的真实情绪。”（P3-1）
- “它可以感受到我个人的情绪，利用音乐帮助我调节情绪和调整心情，很少有 AI 能做到，让我惊喜。”（P3-2）

**B · 系统设计 / 新颖性**
- “很新颖很有趣很丰富的设计，让我感受到 AI 对人类情绪感知的推测和改善功能。”（A-P01）
- “多维度进行分析，互动性强。”（M2-P01）
- “形式简单，效果好。”（M2-P02）

**C · 隐私 / 交互自然度**
- “在最大程度上给了被试尽可能自然的环境，接近真实体验。”（A-P01）
- “互动性强，由个体自身控制，很有意思。”（M2-P01）
- “非常便捷。”（M2-P02）
- “挺好的。”（P3-1）
- 自动模式认可：“自动模式比较自然流畅，不需要专门打开并点击录制，系统能自动拾取我的情绪表达。”（P3-2）

### 偏好理由（为什么选某段音乐）

- 偏好平静陪伴感：“第一段有平静的陪伴感、基调平和偏上、符合需求；第二段过于舒缓悲伤。”（A-P01，选 ToM）
- 偏好舒适专注：“a 更加舒适，让你更集中。”（A-P02，选 ToM）
- 偏好舒缓契合：“节奏比较舒缓好听符合平静的心理。”（A-P03，选 ToM）
- 偏好钢琴音色：“个人主观更喜欢钢琴的音色，更加平静。”（M2-P01，选 Std）
- 偏好轻松节奏：“节奏更轻松愉悦。”（M2-P02，选 Std）
- 偏好整体协调：“整体的协调感。”（P3-1，tie）
- 偏好丰富变化：“节奏、音调变化更丰富，更有助于调整疲惫状态、让我兴奋起来。”（P3-2，选 ToM）

### 改进诉求

**D · 音乐质量 / 丰富度**（3 人，最集中的改进点）
- “音乐的丰富度可以提升，比如乐器节奏。”（A-P01）
- “音乐不够优秀。”（A-P02）
- “音调、节奏可以更丰富，听起来更有趣、更接近人创造的作品。”（P3-2）

**E · 生成延迟**（2 人）
- “缩短一下等待时长。”（M2-P01）
- “处理时长简短。”（M2-P02，希望更短）

**F · 交互方式 / 录制模式**
- “手动模式下点击录制稍微有点奇怪——情绪不好时本会自然表现在语言、表情上，点击录制一定程度上打断了这种表达。”（P3-2）

**G · 输出形式**
- “希望有文字而非纯音乐。”（P3-1）

## 5. 主要结论

1. **核心价值成立**：感知准确率高（H1）、盲评多数偏好 ToM 推理（H3 边际支持），用后效价平均 +1.0、疲劳普遍下降（H4）。
2. **音乐质量是首要短板**：这是唯一 Std ≥ ToM 的维度——ToM 更丰富的推理有时反而产出不够平稳的曲子；多位被试（A-P01、A-P02、P3-2）明确希望**音色/节奏更丰富、更像人创作**。曲风稳定性与丰富度需并重优化。
3. **延迟与交互摩擦**：Q6 延迟接受度是最低项之一（均值 3.75）；ToM/Std 各约 57–74s 生成等待被反复点名（M2-P01、M2-P02）；P3-2 指出**手动模式点击录制会打断自然情绪表达**，自动拾取更自然。
4. **隐私可接受**：Q5 均值 4.13，无强烈抵触。
5. **局限**：N=8、便利抽样、单次体验、18–22 女性为主；感知 ground truth 依赖自评（A-P04 即典型的自评—系统分歧）。结论为趋势性、不可泛化。
